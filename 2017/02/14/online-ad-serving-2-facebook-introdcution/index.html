網路廣告投放（二）Facebook 廣告平台 - 介面及功能簡介篇

發表於 2017-02-14   |   分類於 marketing   |   0 comments   |  

這篇要跟大家介紹的是 Facebook 廣告平台的介面及功能簡介，了解了這些功能之後我們才能慢慢解釋廣告投放的影響因素是什麼。

Facebook 廣告平台 - 介面及功能簡介篇：

首先我們先進入 Facebook 的商業後台及廣告管理員，粉專跟企業平台的綁定這邊就不多贅述了。我們直接進到建立廣告的頁面來解釋 Facebook 後台。

Facebook 目前建立廣告的第一個頁面大致上會長這樣：

左側邊這直排我們可以看到 Facebook 將網路廣告分為三個層次（幾乎每家系統都長這樣啦），依序為 Campaign（行銷活動）、Ad Set（廣告組合）、Ads（廣告）。那先稍微解釋一下每個層次能更改的條件式有哪些好了。

  • Campaign（行銷活動）：

    這個層次稱為行銷活動，其實就是選訂你要的目標（其實是廣告收費方式）啦！在投放任何廣告之前，我都會先問問自己，我這次要追求什麼樣的目標？但這是自己知道就好，因為你選的任何一個 Facebook 上的目標都只是一個「參考」，這個選定的目標會依照 Facebook 的演算法邏輯去投放我們的廣告，但實際上對我們而言，最後攸關到的其實是「廣告的功能、樣式、及位置」以及「廣告的計價方式」，嚴格來說最後的目標成效其實得靠自己去比對的。如果要完整查看每個行銷活動影響到的計價方式的話，我推薦這篇《Facebook行銷活動與競價對應表》，用表格的方式寫得十分詳細！

    B.T.W 行銷活動中其中有幾個目標是需要埋放 Facebook Pixel（ FB 像素）程式碼，才能達到優化效果的，所以在沒有程式碼協助的情況下，基本上是激發不出效果的。

  • Ad Set（廣告組合）

    在一個行銷活動之中，可以放置數個廣告組合，而在廣告組合這一層呢，基本上我們能去調整的東西叫做「受眾」、「版位」、「預算」。有關「受眾」的設定其實就是 Facebook 廣告平台最強大的元素之一，因為 Facebook 可以輕易蒐集到每位使用者的基本資訊跟使用行為並將其分類，所以也讓我們在投放廣告時可以輕易投放到我們設想的目標受眾之中；而「版位」就可以去設定像是裝置類型（一般桌面網頁、IOS、Android ）、或是 Facebook 旗下的哪些平台（ Facebook、Instagram、Messenger 或是旗下的 App 聯播網），而能投放的版位也會因為隨著上一層選的廣告目標類型不同，而有所限制；最後「預算」則是控制著這個單一的廣告組合，每天或每組將花費多少的預算上限，根據投放的額度也會決定廣告的短期流量，除此之外也可以設置廣告的時間排程，而預算中最重要的是，也可以提出預計所花費的 Bid 廣告競價金額，可以選擇的收費方式會依照上一層所設置的目標類型不同，而有算變化。
    整體來說，廣告組合會大大地影響我們廣告最後的表現。

  • Ad（廣告）

    廣告這一層，顧名思義就是在真正將我們設計好的廣告放上去的地方囉！一個廣告組合之中能放置數個廣告，每個廣告都可以設計不同的文案、格式（單一圖片、輪播圖片、影片、全螢幕互動）、廣告點擊連結、以及不同的點擊按鈕。這裡我們除了可以設置各種不同的廣告文案及外觀之外，最重要的是我們可以在廣告點擊連結之中加入追蹤參數，藉此辨別各個廣告的最終成效，除了使用 Facebook 的參數設定之外，或是我們可以直接使用 Google URL Builder 把網址加上 UTM 規則（這篇文章講得蠻細的可以直接參考）的追蹤參數，方便直接從第三方的 Google Analytics 追蹤成效。

前面雖然講完了大致上 Facebook 提供給我們的介面功能，但我還是得稍微詳細講一下「Audience（廣告受眾）」及「 Facebook Pixel」的功能：

  • Audience（廣告受眾）：
    除了設定「年齡」、「地點」、「性別」、「興趣」、「關係鏈」等預先選項之外，我們還可以直接利用「使用者行為」建置更符合明確受眾。

    Facebook 強大的受眾資料庫裡面有超級多的選項可以提供給我們發揮，而「 使用者行為」一開始可以選擇的包括「顧客檔案」、「網站流量」、「應用程式動態」、「在 Facebook 上的互動」。

    「顧客檔案」其實就是可以將你已經有的會員資料像是 Email、Facebook 用戶編號、年齡、性別等資料直接輸入 Facebook 整合。而「網站流量」、「應用程式動態」就是可以根據 Facebook Pixel 或 Facebook Plugin 去追蹤使用者在你的網站或應用程式上的行為，像是「有留言過某篇文章的人」、「有將商品加進購物車行為的人」、「有瀏覽某些商品種類的人」，條件式會根據我們埋放不同的 Pixel 或 Plugin 有所不同。

    而「廣告受眾」其實才是自操還有委託技術廠商的最大差別，技術廠商可以寫出更細緻的特定行為， 且其擁有的使用者資料來源，是我們自操所無法比擬的（例如我們的競爭對手也是他的客戶，那搞不好可以直接使用競爭對手的訪客來源）。

  • Facebook Pixel （FB 像素）：

    Facebook Pixel 其實是 Facebook 在網路界制霸天下最可怕的武器之一。Facebook 除了我們一般人知道的，會搜集我們在動態上的各使用者行為之外，也藉由「有埋入 Pixel 的全球網站之中」，默默地在該網站對追蹤瀏覽者的行為，並給予瀏覽者相對應的編號，然後再將其編號和有登入 Facebook 用戶留下的 cookies 做比對，這就是 Facebook 得以搜刮全世界的使用者行為資料並存在資料庫中加以分析的秘密！（ 而 Google 也是大同小異啦 ）

    直接擷取臉書的官方畫面潮一下：

    那對於我們投放廣告的人而言，我們其實只需知道，基本上 Facebook Pixel 就是一段制定好的 JavaScript 程式碼，將這 Pixel 程式碼放入我們網站相對應的位置之中，它就可以協助我們三件事情：「建立受眾」、「追蹤成效」及「優化成效」。

    1. 建立受眾：

      簡單來說，上一點我們提到的 Audience（廣告受眾）就是建立在 Pixel 的基礎下，所以我們才能輕易地使用像是依據「使用者行為」及甚至是依據「興趣」所區別的廣告受眾。

    2. 追蹤成效：

      在 Facebook 的廣告報表後台上，我們可以在自訂欄位之中看見『轉換次數』這個大項目，其中所有的小項目都是得安裝 Pixel 才能確實追蹤的數據，像是「購買」、「註冊」..等等。也就是說如果沒有安裝 Pixel ，其實我們就無法直接從後台看出廣告實際帶來的轉換效益為何，除非我們用其他第三方工具配合，像是前面有提到的 GA + URL Builder。而實際的做法我們會在後面的章節提到。

    3. 優化成效：
      記得我們前面曾經提到，有些行銷活動的目標必須要有 Pixel 才可以激發出效果嗎？那就是因為 Pixel 達到追蹤成效之後，我們才可以根據實際成效去優化我們的廣告效果以及將投放成本壓低。而實際的做法我們也會在更後面的章節提到。

    至於 Pixel 的埋放安裝流程可以直接參考《 Facebook 的官方教學 》。另外，我們也可以使用 Facebook Pixel Helper 這個 Chrome 插件來幫我們檢查網站上的每頁 Pixel 是否有安裝正確！

我們終於大致將 Facebook 的平台簡介完囉（比我想像中還多許多囧）！接下來，再開始實戰前，我們還得先瞭解一下實際在 Facebook 上的競價機制為何（實際上瞭解一套平台的競價機制後，基本上可以適用大部分的平台），在還沒瞭解 Bid 廣告競價機制之前，我們也沒辦法開始解釋優化的邏輯是基於什麼理由，所以就讓我們來一探這個較少行銷人有研究的地方吧！
